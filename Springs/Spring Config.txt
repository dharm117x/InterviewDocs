==========
>We have a bean that should only be active during development but not deployed in production.

// getting active prifile name.

@Value("${spring.profiles.active}")

@Component
@Profile("dev") or @Profile("!dev") // exclude dev env

Profile can be set in varius way:
1. JVM paramenter: -Dspring.profiles.active=dev
2. Web.xml:
<context-param>
    <param-name>spring.profiles.active</param-name>
    <param-value>dev</param-value>
</context-param>

3. maven profile
	pom.xml : 
	<profile>
        <id>prod</id>
            <properties>
            	<spring.profiles.active>prod</spring.profiles.active>
             </properties>
    	</profile>

	Applcaition.properties: spring.profiles.active=@spring.profiles.active@
	mvn clean package -Pprod

4. linux: export spring_profiles_active=dev // via env variable.

@Transactionl: manage transactions declaratively.
------------------------------------
When you annotate a method with @Transactional, Spring creates a proxy of that class. 
The proxy wraps your method call with transactional logic — like starting, committing, or rolling back a transaction.

Spring uses either:
JDK Dynamic Proxies: If you’re calling an interface
CGLIB Proxies: If it's a concrete class

Note: spring.aop.proxy-target-class=true  # Forces CGLIB proxies
System.out.println(AopUtils.isAopProxy(myService)); // true
System.out.println(myService.getClass());


Default Behavior:
-------------------
Rollback for: RuntimeException and Error

No rollback for: Checked exceptions (Exception) unless specified

Isoaltion: Isolation.DEFAULT, Isolation.READ_COMMITTED(Oracle) 
// Spring does NOT set any special isolation level by itself — it uses whatever the database’s default isolation level is

Propagation:  Propagation.REQUIRED // If there's an existing transaction, join it. If there's no transaction, start a new one.


Customize with:
----------------
@Transactional(rollbackFor = Exception.class) // Rollabck for checked exception.
@Transactional(propagation = Propagation.REQUIRES_NEW, isolation = Isolation.SERIALIZABLE, rollbackFor = {SQLException.class})
@Transactional(readOnly = true) //  going to read data. No inserts, updates, or deletes and reading becomes faster and safer.

REQUIRED (default): Use existing transaction or start a new one.
REQUIRES_NEW: Suspends any existing transaction and starts a new one.

Isolation: levels control how changes made by one transaction are visible to others:

@Transactional(isolation = Isolation.READ_COMMITTED)
public void doSomething() {
    // Prevents dirty reads
}

@TransactionalEventListener : 
It’s used for listening to Spring events within a transactional context, and you can control when the listener is executed in relation to the transaction.

Ex:  You want to send a confirmation email only if the order is saved successfully in the DB. 

@Service
public class OrderService {

    @Autowired
    private ApplicationEventPublisher publisher;

    @Transactional
    public void placeOrder() {
        // save order to DB
        Long orderId = 123L; // pretend it's saved

        // publish event
        publisher.publishEvent(new OrderPlacedEvent(orderId));
    }
}

@Component
public class OrderListener {

    @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
    public void handleOrderPlaced(OrderPlacedEvent event) {
        // This will only run AFTER the transaction is successfully committed
        System.out.println("Sending confirmation for order ID: " + event.getOrderId());
    }

    @TransactionalEventListener(phase = TransactionPhase.AFTER_ROLLBACK)
    public void onFailure(OrderPlacedEvent event) {
    // log or notify failure
    }	

}

AFTER_COMPLETION: Runs after commit or rollback
BEFORE_COMMIT: Runs right before committing

Note: Works only if the event is published within a transactional method.
